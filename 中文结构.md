

下面按**现在实际文件结构**讲。先给一句总定位：

```text
app     = 应用总控、页面切换、全局状态、存档
core    = 战斗棋盘底层数据结构，也就是 Board / Hex / default_board.json
combat  = 战斗相关逻辑、战斗场景、buff/effect/equipment
world   = 宏观世界内容，目前主要是三层路线地图
gui     = 所有界面显示，包括页面、HUD、战斗棋盘 UI item
entity  = 游戏实体，角色、单位、组件、物体、障碍
assets  = 美术资源，宏观地图背景在 assets/maps
```

**入口和应用层**

[main.cpp](D:/project/Synera_starter/src/main.cpp)  
程序入口。只负责创建 `QApplication` 和 `AppWindow`。这个现在是对的，入口不应该直接进战斗页。

[appwindow.h/.cpp](D:/project/Synera_starter/src/app/appwindow.cpp)  
主窗口。内部用 `QStackedWidget` 管 `StartPage / MapPage / EventPage / BattlePage`。它负责连接页面信号和 `GameManager` 信号。  
目前合理，但之后如果页面更多，可以把连接逻辑拆一部分出去。

[gamemanager.h/.cpp](D:/project/Synera_starter/src/app/gamemanager.cpp)  
游戏流程中枢。负责新建游戏、读档、保存、进入节点、事件结束、战斗结束、层推进。  
这是现在最重要的类，定位是对的。

[gamestate.h/.cpp](D:/project/Synera_starter/src/app/gamestate.cpp)  
全局游戏状态和存档数据。保存 seed、生命、金币、当前层、当前节点、地图、已完成节点、可点击节点。  
目前够用，但后面要继续加：背包、角色持有列表、商店状态、事件状态、奖励状态。

[savemanager.h/.cpp](D:/project/Synera_starter/src/app/savemanager.cpp)  
只负责 JSON 存读档。现在支持 3 个槽位。  
定位正确，不应该管页面，也不应该管流程。

**现在不对或还不够清楚的地方**

1. `BattleScene` 还承担太多  
它现在既管 UI 场景，又管准备阶段，又管战斗副本。短期能用，长期建议：
```text
BattlePage     = 页面
BattleScene    = QGraphicsScene 显示和拖拽
BattleSystem   = 战斗规则
BattleContext  = 一场战斗的数据
```

2. `BattleSystem` 还没接上  
现在点击 Finish Battle 就算胜利，占位可以，但真正战斗阶段还没开始。

3. 缺 `BattleContext / BattleResult` 的正式数据结构  
现在 `BattleResult` 是简单 struct，放在 `gamemanager.h` 里。后面应该移到 `combat/battlecontext.h` 或 `combat/battleresult.h`。

4. 缺事件系统  
现在没有：
```text
world/event/event.h
world/event/eventmanager.h/.cpp
```
所以 `EventPage` 只是占位。

5. 缺真实 Bag / Shop  
`GameHud` 里只是弹窗占位。后面需要：
```text
entity/item/
app/playerstate 或 inventory
gui/widgets/bagpanel
gui/widgets/shoppanel
```

6. `GameState` 还太薄  
现在只存地图进度、HP、金币。后面至少要加：
```text
ownedCharacters
bagItems
shopState
eventFlags
currentRunStatus
```

7. `combat/map_utility` 名字容易误导  
建议改成 `combat/board_utility` 或放到 `core/board_utility`，因为现在项目里有两种 map，名字要狠一点分开。

8. `gui/HUD` 和 `gui/widgets` 命名有点重叠  
`gui/HUD/characterhud` 是战斗单位头顶 HUD，`gui/widgets/gamehud` 是页面公共 HUD。语义能分开，但目录名建议统一成：
```text
gui/board_hud/
gui/widgets/
```
或者：
```text
gui/board/overlays/
gui/widgets/
```

**现在最缺的三块**

1. `BattleSystem` 真正接管战斗阶段  
让 `BattleScene` 只负责显示和拖拽。

2. `EventManager`  
让事件页面不再写死占位文本。

3. `PlayerState / Inventory / ShopState`  
把玩家生命金币、角色、背包、商店从 `GameState` 里拆成更清楚的子结构。

目前整体方向已经顺了：**Board 是战斗棋盘，Map 是宏观路线图；core 是底层棋盘模型，不再当杂物间。**
