# Synera: Synergy Auto-Arena

Synera 是一个使用 C++17 和 Qt 6 制作的单机 PVE 自走棋课程作业。项目灵感来自自动战斗、路线探索和 Roguelike 构筑：玩家带领一支小队穿过三层路线地图，在事件、休息、海克斯科技和战斗之间做选择，通过招募、升星、羁绊、装备和站位构筑最终阵容。

这份 README 把项目当作一个游戏来介绍，重点说明玩法、系统设计、运行方式和打包内容。

## 游戏玩法

### 核心循环

1. 在路线地图上选择一个可达节点。
2. 根据节点类型进入普通战斗、精英战斗、事件、休息或 Boss。
3. 战斗前在备战区调整队伍、购买角色、穿戴装备和安排站位。
4. 开始战斗后单位自动索敌、移动、攻击和释放技能。
5. 战斗胜利后获得金币、装备或事件奖励，继续推进路线。
6. 击败三层 Boss 即通关；生命值归零则本局失败。

### 阵容经营

- 角色卡：商店提供 5 个随机招募位，购买后加入队伍。
- 升星：同名同星角色可以合成更高星级，提升属性。
- 上阵限制：战斗前可拖拽角色到棋盘，超过当前上阵上限时不能继续布阵。
- 回收：不需要的角色可从备战区回收，返还部分金币。
- 记录：游戏会记录通关或失败的跑团结果。

### 战斗

- 棋盘使用六边形格子。
- 玩家角色和敌人都使用统一的 `Character` 战斗模型，通过队伍归属区分敌我。
- 单位会自动寻找敌人；在攻击范围外会绕开障碍和已有单位移动。
- 普攻积累法力值，法力满后自动释放技能。
- 角色技能通过 Buff/Effect 触发，可以在战斗开始、攻击前后、受击前后、死亡前和时间 tick 中生效。

### 构筑系统

- 羁绊：角色拥有战士、法师、建筑、魔女、刺客等标签，上阵达到阈值后触发团队加成。
- 装备：战斗和事件会掉落基础装备，装备可以穿戴到角色身上。
- 装备合成：基础装备可以按配方合成为高级装备，形成更强的属性组合。
- 海克斯科技：每层开始可选择一张海克斯卡，提供长期 Buff、资源奖励或路线/掉落修正。
- 事件：路线中的事件节点提供风险收益选择，部分事件会触发战斗并在胜负后回到事件文本。

## 系统设计

项目分为六个主要层次：

```text
src/app      游戏流程、存档、窗口切换、事件和战斗回流
src/world    路线地图、事件、海克斯科技、背包/商店状态
src/combat   战斗系统、战斗配置、Buff/Effect、羁绊、装备
src/core     Hex 坐标和战术 Board
src/entity   Unit、Character、Object、Obstacle 和角色组件
src/gui      Qt 页面、战斗场景、HUD、卡牌面板、装备面板
```

主要类：

- `GameManager`：游戏流程中枢，管理地图推进、事件动作、商店、装备、战斗和存档。
- `GameState`：可序列化的全局状态，保存生命、金币、地图、角色卡、装备、商店、海克斯和当前事件。
- `Board` / `Hex`：战斗棋盘和六边形坐标，负责地块、占位、移动、邻居和距离。
- `Character`：可战斗单位，由属性、攻击、移动、队伍、Buff、渲染、装备等组件组合。
- `battlesystem`：战斗规则系统，负责加载战斗、生成敌人、应用羁绊/装备/海克斯、推进战斗和结算。
- `BattleScene`：Qt 战斗棋盘控制器，负责备战区、拖拽布阵、棋盘图元、障碍物、宝箱和动画同步。

更完整的架构说明见 `architecture.pdf`。

## 构建运行

环境要求：

- Qt 6，包含 `Core`、`Gui`、`Widgets`
- CMake 3.16 或更高版本
- 支持 C++17 的编译器

构建示例：

```bash
cmake -S . -B build
cmake --build build --config Release
```

构建完成后，CMake 会把运行所需的棋盘、事件、战斗、装备、海克斯配置和 `assets/` 复制到可执行文件目录。
